# RT-Thread AM BSP Port

Port of RT-Thread to the Abstract Machine BSP at `workloads/rt-thread-am/bsp/abstract-machine`, targeting `ARCH=riscv32e-npc`.

## Build

```bash
cd workloads/rt-thread-am/bsp/abstract-machine
export AM_HOME="$(cd ../../../abstract-machine && pwd)"
export ARCH=riscv32e-npc
source ../../../../data/workload-build/config.sh
make init   # required after .config changes or first build
make -j4
make insert-arg
```

## Run on emulator

```bash
"$AM_HOME/../../emulator/build/emulator" -e "load build/rtthread-riscv32e-npc.bin 0x20000000; run 2000000; exit"
```

## BSP changes

- Enabled `RT_USING_LIBC` and added minimal freestanding headers under `include/`:
  `sys/types.h`, `sys/errno.h`, `time.h`, `ctype.h`, `wchar.h`, `errno.h`,
  `fcntl.h`, `string.h`, `unistd.h`.
- Restored DFS in `.config` (the default had it enabled); switched from `elm-fat`
  to `ramfs`/`romfs`/`devfs` to match the working AM port and avoid the heavy
  FAT/SD block-device dependencies.
- Added `RT_USING_MEMHEAP` because `ramfs` depends on it.
- `rtconfig.h` is now regenerated by `make init` whenever `.config` changes
  (`Makefile` dependency fix).
- `Makefile` puts the BSP `include/` directory before AM/RT-Thread headers so
  our POSIX declarations are picked up, and adds the RT-Thread libc extension
  include path needed by `sys/stat.h`.
- Removed the broken Python one-liner that tried to inject `#include "extra.h"`
  into `rtconfig.h`; `extra.h` is empty for non-native targets anyway.
- Removed `RT_DEVICE_FLAG_INT_RX` from the console UART registration path in
  `components/drivers/serial/serial.c` so the shell polls `_uart_getc()`.
- Implemented context switching in `src/context.c` via AM `yield()`; the yield
  event handler advances `mepc += 4` so the switched-out thread resumes after
  the `ecall`.

## Linker script

`extra.ld` places `FSymTab`, `VSymTab`, `.rti_fn.*`, and `UtestTcTab` into
`.rodata.extra`. `ld.lld`'s `SORT(.rti_fn*)` does not order the init levels
correctly, so the sections are listed explicitly:

```ld
KEEP(*(.rti_fn.0))
KEEP(*(.rti_fn.0.end))
KEEP(*(.rti_fn.1))
KEEP(*(.rti_fn.1.end))
KEEP(*(.rti_fn.2))
KEEP(*(.rti_fn.3))
KEEP(*(.rti_fn.4))
KEEP(*(.rti_fn.5))
KEEP(*(.rti_fn.6))
KEEP(*(.rti_fn.6.end))
```

## Makefile fix

`workloads/abstract-machine/scripts/riscv32e-npc.mk` resets `LDFLAGS` with `:=`,
so setting `LDFLAGS += -T extra.ld` before `include $(AM_HOME)/Makefile` was
silently dropped. The BSP Makefile now appends both after the include:

```makefile
include $(AM_HOME)/Makefile
LDSCRIPTS += extra.ld
LDFLAGS += -T extra.ld
```

## `halt` and shell command sequence

- `src/halt.c` exports `halt` as an MSH command that calls AM `halt(0)`.
- `src/uart.c` feeds a predefined command sequence so the emulator stops cleanly
  after exercising the shell:

```c
static const char *p = "help\ndate\nversion\nfree\nps\nutest_list\n"
                       "pwd\nls\nmemtrace\nmemcheck\nhalt\n";
```

All commands execute successfully:

- `pwd` prints `/`.
- `ls` reports `No such directory`, which is expected because no filesystem is
  mounted at the root directory.
- `memtrace` dumps the heap breakdown.
- `memcheck` is silent on success (it only prints when it finds corruption).
- `halt` stops the emulator cleanly.

## Alignment fix in `ctime.c`

`components/libc/compilers/common/ctime.c` reads the `days` and `months` strings
through `*(int*)` casts in `asctime_r`. The original declarations only aligned
the pointer variable, not the string literal it points to. In this BSP's rodata
layout the literal ended up at an address that is not 4-byte aligned, so the
`lw` in `asctime_r` raised `mcause=4` (load address misaligned) and the shell's
`date` command panicked.

Fixed by storing the strings in separately aligned arrays:

```c
rt_align(4) static const char days_data[] = "Sun Mon Tue Wed Thu Fri Sat ";
static const char *days = days_data;
rt_align(4) static const char months_data[] = "Jan Feb Mar Apr May Jun Jul Aug Sep Oct Nov Dec ";
static const char *months = months_data;
```

## Verification

Emulator output shows the RT-Thread banner, `Hello RISC-V!`, the `msh />`
prompt, and successful execution of `help`, `date`, `version`, `free`, `ps`,
`utest_list`, `pwd`, `ls`, `memtrace`, `memcheck`, and finally `halt`.

The same scripted workload also passes unified RTL difftest:

```bash
../../../../emulator/build-release-rtl/emulator-rtl --difftest \
  -e "load build/rtthread-riscv32e-npc.bin 0x20000000; run 2000000; exit"
```

It halts cleanly after 567,442 matched retired instructions.
